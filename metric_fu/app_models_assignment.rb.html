1# Assignment is the task/deliverable on which the professor grades for students.
  
2#
  
3# Professor can create/view/modify assignments by going to manage assignment tab in course tools, on the index page of
  
4# each course. He/She can also limit submission from student, can change the type of assignment and submission date if
  
5# required. The assignments will automatically get sorted out based on the course and the task. Student will not be able
  
6# to modify the assignment, only faculty can do this. The assignment could be graded by points/weights/letter, and it
  
7# can be modified in the Course Configuration page.
  
8#
  
 9# Generally, a course has many assignments, and each assignments contains many student grades and many submitted deliverables.
  
10#
  
11# * Professor can create an assignment on course assignments index page by clicking the "New Assignment" link.
  
12# * Assignment can be grouped by task_number.
  
13# * Assignment has default scope to get ordered based on task_number and assignment_order
  
14# * assignment_order is auto generated by system, professor cannot directly specify the number of assignment_order. Instead, they can use drag-drop function to reorder them.
  
15# * task_number, due_date, name can be blank
  
16# * maximum_score is a required field. Professor should give a maximum score (>=0 ) for each assignment.
  
17# * is_team_deliverable tells the assignment is a team deliverable or individual deliverable
  
18# * is_submittable is designed for those assignment that don't required any submission, e.g., course participation, effort log.
  
19# * verify_deliverables_submitted tells whether there are any deliverable submmited for this assignment. If so, the professor could not delete this assignment.

102#### PGSQL PG::Error
  
103
  
104On running through the steps, sometimes you might encounter this error:
  
105
  
106    PGSQL PG::Error (could not connect to server: No such file or directory
  
107      Is the server running locally and accepting
  
108      connections on Unix domain socket "/var/pgsql_socket/.s.PGSQL.5432"?
  
109    ):
  
110
  
111This is happening because OSX (Mountain Lion) comes up with a version of Postgres pre-installed. This is getting picked up instead of the one you manually installed. Doing one of the below should resolve the issue:
  
112
  
1131. Add /usr/local/bin to the $PATH directory
  
114
  
115If you used Homebrew for installation, then a symbolic link to the postgres executable is placed in the directory /usr/local/bin. Ensure it's there and then add that directory your PATH environment variable.
116
  
116
  
1172. add a specific host to your database.yml
  
118
  
119Your database.yml should look similar to the below:
  
120
  
121    development:
  
122        host:     localhost
  
123        adapter:  postgresql
  
124        encoding: utf8
  
125        database: cmu_education
  
126        ...

158### Compiling and deploying assets to Heroku:
  
159
  
160(The following commands and rakes assume that you are already using asset_sync gem and that it is configured correctly):
  
161
  
1621. Delete any assets found under the public folder: bundle exec rake assets:clean
  
1632. Compile new assets to generate/update the manifest.yml: bundle exec rake assets:precompile:all RAILS_ENV=production , heroku needs the production environment. You can change this to development if you want to make a quick test. But be ware that you can not predict the full assets behaviour until you test it on the production. This command might take some time, so be patient.

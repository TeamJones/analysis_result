1# Peer Evaluations are done by student teams at the request of a faculty member.
  
2# Usually, the faculty member will setup the peer evaluation and will need to
  
3# know the following information
  
4# * Start date - when to email the student team about the evaluation.
  
5#   Teams can start before this
  
6# * End date - when to email the student team
  
7#   Teams can end after this provided the faculty hasn't created the report.
  
8# * Learning Ojbectives for each team member (optional)
  
 9#

16#
  
17# == Assumptions
  
18# * The peer evaluation for a team is always ready, theoretically a team could
  
19#   start one on their own
  
20# * A team would not need to do a peer evaluation twice. If they had to,
  
21#   a system admin would clean out the data or alter the foreign keys to appear
  
22#   as if a peer evaluation had not been done for that team
  
23# == Security
  
24# * Students should not be able to see the peer evaluations for other students
  
25# * Any faculty can see the peer evaluation for a team
  
26#
  
27# This component was originally written by Russel Reed (Class of 2010)
  
28# and integrated by Todd Sedano. Student teams can provide 360 review feedback
  
29# when prompted by the faculty.
